# 小红书市场需求挖掘智能体 (XHS Needs Discovery Agent)

这是一个基于 LangGraph 与大语言模型（LLM）构建的自动化需求分析智能体应用。它通过结合 `Spider_XHS-master` 所提供的底层小红书 API 接口，实现全自动化地搜索爆款笔记、抓取评论区、然后经由 LLM 剔除闲聊和水军，提炼出对特定领域最核心的需求与痛点，最终整理输出详尽的"市场痛点与产品需求调研报告"。

## 1. 核心特性

- **纯内存交互**：跳过了多媒体音视频资源的下载和本地 Excel I/O 写出，大幅度提高爬取速度。
- **智能筛选去重**：长达几百条的评论会被送进 LLM 的漏斗中，LLM 会过滤无关内容，只将"求平替"、"抱怨吐槽质量"、"求功能"之类的实际有效诉求剥离出来。
- **自动报告生成**：每次扫描总结后会自动将结果汇总写出 MD 格式的学术级调研报告，存于 `reports/` 目录下。
- **可配置抓取数量**：可在 Web 界面或 CLI 中自由设置每次调研抓取的帖子数量（1~20 篇），默认 3 篇，数量越多结果越全面，耗时也越长。

## 2. 准备运行工作

### 2.1 环境变量配置

由于此流程严重依赖网页版小红书的合法身份，您需要先去完成下列信息的配置工作。

1. **配置 OpenAI 秘钥：** 在您的终端中或 Pycharm 设置好您的 `OPENAI_API_KEY`（或其它大模型密钥）。
2. **提取有效 Cookie：**
    - 在浏览器无痕模式或日常登录状态进入 `https://www.xiaohongshu.com/explore`。
    - 按下 `F12`，捕获抓包接口中的带有登录身份证明的 `Cookie` 字段。
    - 前往克隆进项目根目录的 `Spider_XHS-master/` 中找到或创建 `.env` 文件。
    - 将刚刚的 Cookie 直接放入该 `.env` 里，格式如下（可使用 COOKIE / COOKIES 或 cookies）：
      ```env
      COOKIES='abRequestId=xxx... loadts=xxx...'
      ```

### 2.2 NodeJS 环境

因为 `Spider_XHS-master` 包的接口签名破解依赖 JS 提供算法计算（借助 `execjs`），您必须保证自己的环境中装配好了依赖。
确保您已经在该爬虫模块下成功执行过：
```bash
cd Spider_XHS-master
npm install
```

## 3. 运行方法与指引

### 3.1 Web 界面（推荐）

启动后端服务：
```bash
python server.py
```
或：
```bash
uvicorn server:app --host 0.0.0.0 --port 8000
```

打开浏览器访问 `http://localhost:8000`，即可使用完整的 Web 界面：

- 在搜索框输入调研话题（如"降噪耳机"）
- 通过 **`−` / `+`** 按钮调整抓取帖子数量（1~20 篇，默认 3 篇）
- 点击「开始调研」，实时查看 Agent 各节点执行进度
- 调研完成后直接在页面阅读或下载 Markdown 报告
- 左侧侧边栏可随时查看和重新打开历史报告

### 3.2 命令行（CLI）

退回项目**根目录 (`RedNoteGet`)** 下，执行调用 `main_agent.py`，参数 `--query` 是您的痛点挖掘对象：

```bash
python -m agent.main_agent --query "降噪耳机"
```
```bash
python -m agent.main_agent --query "笔记本支架"
```
```bash
python -m agent.main_agent --query "什么手机发热严重"
```

> CLI 模式下抓取数量固定为 3 篇。如需调整，可修改 `agent/nodes.py` 中 `post_searcher_node` 的默认值，或在调用 `create_agent` 时通过 `initial_state` 传入 `post_count` 字段。

## 4. 抓取数量说明

| 帖子数量 | 预计耗时 | 适用场景 |
|----------|----------|----------|
| 1~2 篇   | 约 1 分钟 | 快速预览，验证关键词效果 |
| 3 篇（默认） | 约 2~3 分钟 | 日常调研，结果较为均衡 |
| 5~10 篇  | 约 5~10 分钟 | 深度调研，覆盖更多用户声音 |
| 10~20 篇 | 约 10~20 分钟 | 全面分析，适合正式产品决策 |

每篇帖子最多采集 150 条有效评论（过滤掉少于 4 字的无效内容）。

## 5. 报告结果查看

执行过程通常花费 1 到 5 分钟不等（视抓取篇数和评论量而定）。
完成时命令行会直接印出长报告。随后项目根目录会自动生成 `reports/` 目录，您可以在里面查找到时间戳命名的 `MD` 最终调研产出物。
